# 格式审计组 (Standardization Auditor Agent) 测试说明文档

本文档旨在指导开发人员和测试人员如何验证 Standardization Auditor Agent 的功能，包括环境准备、单元测试、集成测试及手动验证流程。

## 1. 环境准备 (Prerequisites)

在运行任何测试之前，请确保已完成以下准备工作：

1.  **Python 环境**: Python 3.10.x
2.  **依赖安装**:
    ```bash
    cd Standardization_Auditor_Agent
    pip install -r requirements.txt
    ```
3.  **数据库**:
    - 启动 PostgreSQL 服务。
    - 运行自动建库脚本 (包含创建库、安装 `pgvector` 扩展、创建表结构)：
      ```bash
      python ensure_db.py
      ```
4.  **环境变量**:
    - 在 `Standardization_Auditor_Agent` 目录下创建 `.env` 文件。
    - 填入 `QWEN_API_KEY` / `DEEPSEEK_API_KEY` 等配置（参考项目根目录的 `README.md`）。

## 2. CLI 命令行审计测试 (CLI Audit Testing)

这是目前最推荐的测试方式，可以直接对 PDF 文件生成详细的 Markdown 报告。

### 运行方式
```bash
cd Standardization_Auditor_Agent
python main.py --pdf "path/to/your/paper.pdf"
```

### 验证内容
检查 `Standardization_Auditor_Agent/report` 目录下是否生成了以下文件：
1.  **评分报告 (`*_score_report.md`)**:
    -   包含总分（Score）和评级（Critical/Warning/Info）。
    -   包含各类型问题的数量统计。
2.  **扣分细则 (`*_deduction_details.md`)**:
    -   **视觉布局分析 (CV Layout Analysis)**: 检查图表位置、标题层级等布局问题。
    -   **语义内容分析 (LLM Semantic Analysis)**: 检查错别字、标点符号、引用格式等语义问题。
    -   确认 Qwen API 是否成功调用（如果配置了 Key，应能看到语义分析结果）。

## 3. 单元测试 (Unit Tests)

当前仓库可能存在 `tests/` 目录为空或未包含可直接运行的 unittest 用例的情况。此时建议优先使用：
- CLI 命令行审计测试（第 2 节）
- 集成/部署验证脚本 `scripts/verify_deployment.py`（第 4 节）

如后续补充 unittest 用例，可使用以下命令统一运行：
```bash
python -m unittest discover -s tests
```

## 4. 集成/部署验证 (Integration Tests)

集成测试通过启动真实的 FastAPI 服务进程，模拟完整的 HTTP 请求流程，验证系统各模块的协同工作情况。

### 运行方式
在 `Standardization_Auditor_Agent` 目录下运行：
```bash
python scripts/verify_deployment.py
```

### 验证内容
该脚本会自动执行以下步骤：
1.  **启动服务**: 在本地 `127.0.0.1:8001` 启动 API 服务。
2.  **健康检查**: 访问 `/health` 确保服务存活。
3.  **规则加载**: 访问 `/rules` 确保动态规则 (rules.yaml) 已正确加载。
4.  **完整审计流程**: 发送包含 Base64 PDF 的 `/audit` 请求，验证：
    - 响应状态码 200。
    - 返回 JSON 结构符合协议。
    - `result.score` 和 `result.issues` 字段存在。
5.  **数据库回退测试**: 发送不含 `content` 的请求，验证系统是否尝试从数据库拉取内容（预期返回 400；如本地 DB 连接失败则可能返回 500）。
6.  **清理**: 测试结束后自动关闭服务进程。

## 5. 手动验证 (Manual Verification)

### 5.1 服务状态页面
启动服务后，浏览器访问 `http://localhost:8000/` (或配置的端口)。
- **预期结果**: 显示 "Standardization Auditor Agent Status" 页面，包含：
  - 当前版本 (Version)
  - 运行环境 (Environment)
  - LLM 提供商 (LLM Provider)
  - 已加载规则列表 (Loaded Rules)
  - API 文档链接 (Docs)

### 5.2 规则查看接口
浏览器访问 `http://localhost:8000/rules`。
- **预期结果**: 返回 JSON 格式的规则列表，确认 `typo_check`, `heading_check` 等核心规则都在列表中。

### 5.3 数据库验证
连接 PostgreSQL 数据库，查询 `agent_rules` 表：
```sql
SELECT rule_id, content, updated_at FROM agent_rules;
```
- **预期结果**: 应包含 `rules.yaml` 中定义的核心规则（如 `typo_check`, `heading_check`, `figure_table_check` 等）。

## 6. 常见问题 (Troubleshooting)

- **TestClient Error**: 如果遇到 `TypeError: Client.__init__() got an unexpected keyword argument 'app'`，请勿使用 `TestClient` 进行复杂集成测试，直接使用 `scripts/verify_deployment.py`。
- **Database Connection Failed**: 检查 `DATABASE_URL` 环境变量是否正确，以及 PostgreSQL 服务是否运行。
- **Layout Analysis Timeout**: 如果长文档处理超时 (默认 300s)，请在环境变量中增加 `LAYOUT_ANALYSIS_TIMEOUT` 的值。
